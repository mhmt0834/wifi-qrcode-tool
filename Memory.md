## 一、项目使命（Mission）

智联WiFi不是一个简单的WiFi密码分享工具。

它真正要做的是：建立全国最大的线下WiFi入口平台。

让任何一家商铺：

- 上传WiFi
- 获取流量
- 获取广告收益
- 获取客户
- 获取数据分析

最终形成：

商家 → WiFi → 用户 → 广告 → 收益 → 更多商家

形成正向循环。

未来定位不是“WiFi分享”，而是：全国线下流量入口。

## 二、项目最终目标

最终要实现：

全国所有城市 → 所有商家 → 全部上传WiFi → 全部使用智联WiFi

最终形成：中国最大的WiFi数据库。

不仅能连接WiFi，还能：

- 找附近商家
- 看优惠
- 商家营销
- 广告收益
- 会员系统
- AI推荐

未来甚至可以成为：线下版大众点评 + WiFi入口。

## 七、未来最重要架构（永久不能忘）

这是整个项目最重要的设计。

上传人 ≠ 商家

举例：

代理帮商家上传WiFi。

上传人：A  
商家：B

广告收益必须给 B，不是 A。

因此，数据库必须永久分离：

- `creatorOpenid`
- `merchantOpenid`

以后所有收益全部按照 `merchantOpenid` 计算，不是 `creatorOpenid`。

这是以后代理系统的基础，也是以后加盟体系的基础。

这个设计永远不能删除。

## 八、二维码设计

二维码里面不能保存密码。

二维码只能保存：`wifiId`。

扫码以后：

云函数查询数据库 → 判断广告 → 广告完成 → 返回最新密码

这样密码修改以后，二维码不用重新打印，永远有效。

这是整个项目安全的重要设计。

## 十八、长期开发理念（永久原则）

1. 安全优先：所有敏感数据（尤其是WiFi密码）必须由云函数控制，前端和二维码都不能存储密码。
2. 商家利益优先：所有收益归属以 `merchantOpenid` 为准，任何代理或上传者都不能影响商家的收益。
3. 可扩展优先：每新增一个功能，都要考虑未来是否支持多城市、多代理、多门店、多平台。
4. 数据驱动：所有连接、广告、收益、浏览都要有统计，为AI分析和商业决策提供基础。
5. 体验优先：连接流程尽量做到“扫码 → 看广告 → 一键连接”，减少用户操作成本。
6. 全国化目标：从墨玉县验证模式开始，逐步推广到和田、新疆、全国，形成覆盖全国的WiFi网络。
7. 平台化思维：智联WiFi最终不是一个工具，而是一个连接用户、商家、广告、服务的数字化平台。
